## 2025-05-15 - [Portfolio Rendering Optimization]
**Learning:** Cascading re-renders in the portfolio list were triggered by state updates in the parent `PortfolioPage` (lightbox toggle). Even though the lightbox is a separate overlay, the state change caused all `ProjectCard` components to re-render because their props (callbacks) were recreated. Simply adding `React.memo` was insufficient because `cdnOptions` and `customLoader` in sub-components were unstable object/function literals.
**Action:** Always wrap parent callbacks in `useCallback`. Move object literal props (like `cdnOptions`) to stable constants or `useMemo`. Ensure `React.memo` components in this environment have an arrow function definition and explicit `.displayName` to satisfy Turbopack/HMR requirements.
